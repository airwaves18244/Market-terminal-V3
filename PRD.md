# PRD V3 — Market Terminal: аналитический и торговый терминал уровня state of the art

Версия: 3.0 · Дата: 2026-07 · Статус: утверждён к декомпозиции (см. `V3/TODO.md`)

---

## 1. Резюме

Market Terminal V1–V2.5 — десктопный (Tauri) терминал наблюдения за рыночными
потоками (Finam Trade API): оборот/потоки/breadth/RRG/кросс-активы, live
Time&Sales/DOM/алёрты, бэктестер, paper trading, delta/footprint, MOEX ALGO
(Super Candles/FUTOI/HI2/Key Activity), историзация и опционная аналитика —
всё это работает end-to-end на mock/локальных данных, доменная математика
покрыта тестами.

**V3 превращает прототип в рабочий инструмент активного трейдера**, сопоставимый
по ключевым возможностям с эталонами индустрии:

| Эталон | Что берём как планку |
|---|---|
| TradingView | графики, рисование, мультитаймфрейм, скринер, алёрты |
| Bookmap / ATAS / Sierra Chart | heatmap ликвидности, footprint, order flow |
| CQG / MultiCharts | торговый DOM, брекет-заявки, автоматизация стратегий |
| TSLab / Amibroker | walk-forward бэктест, оптимизация параметров |
| Bloomberg (по духу) | единые рабочие пространства, связанные панели, командная строка |
| OpenBB / LLM-копайлоты | естественно-языковой анализ, автоматический market brief |

Не-цели V3: мобильные клиенты, мультиброкерность за пределами Finam+MOEX ISS,
HFT-латентность (<1 мс), социальные функции, крипто.

## 2. Пользователи и сценарии

**P1 — Активный трейдер (день/скальп), основная персона.**
Сценарии: следит за потоками и аномалиями → находит инструмент через скринер /
Mega Alerts → открывает связку график+DOM+footprint → входит брекет-заявкой из
DOM → сопровождает позицию (риск-панель, трейлинг) → журнал сделок.

**P2 — Свинг/позиционный аналитик.**
Сценарии: секторная ротация (RRG), breadth-режимы рынка, кросс-активные
перетоки, кривая доходности, FUTOI-дивергенции; строит опционные структуры под
взгляд; получает утренний LLM-brief.

**P3 — Квант-исследователь.**
Сценарии: качает и валидирует историю (Finam/ALGOPACK), пишет стратегию,
гоняет бэктест с walk-forward и оптимизацией, анализирует роботов-детекторов,
выводит стратегию в paper → live с риск-лимитами.

## 3. Принципы и архитектурные инварианты

1. **Слоистость сохраняется**: вся математика — в `domain` (без API/UI/БД),
   `data`/`storage`/`app` — тонкие адаптеры; кросс-платформенный CI зелёный
   (`fmt`, `clippy -D warnings`, `cargo test --workspace`, svelte-check/vitest).
2. **Тесты прежде интеграции**: сетевые контракты фиксируются фикстурами;
   оркестрация тестируется на фейках без сети.
3. **Mock-first фронт**: каждая панель обязана работать в браузере на моках.
4. **Безопасность по умолчанию**: секреты — keyring/.env (вне репозитория);
   live-трейдинг за явной фичей + двойным подтверждением в UI.
5. **Детерминизм бэктеста**: одинаковые вход/параметры → бит-в-бит одинаковый
   отчёт.

## 4. Функциональные требования

Приоритеты: **M** (must, ядро V3) · **S** (should) · **C** (could, при ресурсе).

### E1 — Данные и подключения (M)

- **E1.1** Боевой live-контур Finam: устойчивый ингест 24/5 — стримы
  quotes/trades/bars с авто-reconnect, деградация в поллинг при сбоях стрима,
  watchdog + метрики фида (lag, gap-детект, счётчик reconnect).
- **E1.2** Транспорт MOEX ISS/ALGOPACK (`data::moex`, фича `moex`): HTTP-клиент
  (`data::http`, retry/backoff/лимиты как в gRPC-слое), эндпоинты tradestats /
  futoi / hi2 / опционная доска; контракт зафиксирован живыми фикстурами
  (блоки `(verify)` из `ROADMAP_PHASE_10-12.md`).
- **E1.3** Историзация production-grade (`storage`): таблицы датасетов DuckDB
  (схема v3) + экспорт/импорт Parquet; загрузчик с планом `missing_ranges`,
  докачкой хвоста, контролем целостности (дубликаты/дыры/монотонность времени);
  прогресс/отмена из UI (`HistoryTab`).
- **E1.4** Качество данных: модуль `domain::quality` — детект пропусков, выбросов
  (аномальные свечи), несоответствий объёма; отчёт качества по датасету в UI.
- **E1.5 (S)** Корпоративные события: сплиты/дивиденды из ISS, поправка истории.

### E2 — Графический движок и рабочие пространства (M)

- **E2.1** Полноценный docking (dockview или эквивалент): свободная раскладка
  панелей, вкладки внутри доков, drag-n-drop, float-окна; сохранение/загрузка
  **именованных workspace** (JSON в настройках) + 3 пресета (Обзор / Скальпинг /
  Исследование).
- **E2.2** Линковка панелей по инструменту и таймфрейму: цветовые группы
  (A/B/C); клик по тикеру в любой таблице переключает все панели группы.
- **E2.3** Продвинутый график (Lightweight Charts + оверлеи):
  мультитаймфрейм-синхронизация, индикаторы (EMA/SMA/VWAP + полосы, Bollinger,
  RSI, MACD, ATR, объёмный профиль сессии), раздельные панели индикаторов,
  инструменты рисования (тренд, горизонталь, фибо, прямоугольник) с
  персистентностью по инструменту.
- **E2.4** Отображение сделок/позиций/заявок на графике (маркеры входа/выхода,
  линии активных заявок с drag-переносом цены — связь с E4).
- **E2.5 (S)** Heatmap ликвидности (Bookmap-стиль): тепловая карта стакана во
  времени из потока DOM-снимков + лента сделок пузырями; Canvas/WebGL-рендер,
  ring-buffer в Rust, децимация до кадра.
- **E2.6 (S)** Кластерный график (footprint chart) как полноценный тип графика:
  bid×ask по ценам внутри свечи, дельта, imbalance-подсветка (диагональный
  дисбаланс ≥ N×), POC/VAH/VAL.

### E3 — Скрининг, алёрты, аналитика (M)

- **E3.1** Скринер: конструктор фильтров по метрикам terminal-wide (оборот,
  %изм, MFI, CVD, unusual volume z-score, HI2, FUTOI-дивергенция, близость к
  экстремумам, спред/ликвидность), сохраняемые пресеты, live-обновление,
  результат — таблица с мини-спарклайнами.
- **E3.2** Единый центр алёртов: объединить `alerts` + Mega Alerts + роботов
  Delta в один поток событий с приоритетами, звуком, тостами, журналом и
  правилом-конструктором (композиция AND/OR/NOT из `keyactivity` — переиспользовать).
- **E3.3** Индикаторный конвейер `domain::indicators`: единый трейт
  инкрементального индикатора (O(1) на бар/тик) — общая база для графика,
  скринера и стратегий; портировать существующие метрики на него.
- **E3.4 (S)** Профиль объёма/рынка: composite/session volume profile, TPO (C).
- **E3.5 (S)** Корреляционная матрица и beta к индексу по вотчлисту.
- **E3.6 (C)** Командная строка (Ctrl+K): «SBER 5m chart», «screener oi>…» —
  быстрый вызов панелей и действий.

### E4 — Торговля (M — paper parity, S — live)

- **E4.1** Торговый DOM production-уровня: одноклик покупка/продажа с лесенки,
  брекет-заявки (вход + SL/TP одним действием, OCO), быстрые объёмы, центровка,
  накопленная глубина, средняя цена позиции на лесенке.
- **E4.2** Типы заявок в симуляторе и роутере: Market/Limit/Stop/StopLimit,
  Day/GTC, OCO/брекет; частичные исполнения; очередь по времени на уровне.
- **E4.3** Риск-движок `domain::risk`: лимиты (макс. позиция/инструмент, дневной
  убыток, макс. количество заявок/мин), автоматический flatten по стоп-лимиту
  дня, kill switch в UI; риск-проверки до отправки любой заявки (paper и live).
- **E4.4** Журнал трейдера: все сделки с контекстом (скриншот-состояние метрик
  на момент входа), P&L-календарь, статистика по инструментам/времени суток/
  сетапам, экспорт CSV.
- **E4.5 (S)** Live-роутинг Finam (`OrderService`/`AccountsService`, фича
  `live-trading`): реальный `FinamOrderRouter` — размещение/отмена/статусы,
  синхронизация позиций и счёта, reconcile при реконнекте. UX-предохранители:
  режимы PAPER/LIVE с явным переключением, подтверждение первой заявки сессии,
  красная индикация LIVE.
- **E4.6 (C)** Полуавтоматизация: исполнение сигналов стратегии с ручным
  подтверждением («стратегия предлагает — трейдер жмёт»).

### E5 — Исследования: бэктест и оптимизация (M)

- **E5.1** Бэктест на тиках/футпринте: движок принимает ленту сделок (не только
  бары) — честное исполнение лимиток по очереди, слиппедж по стакану.
- **E5.2** Оптимизация параметров: grid + random search по диапазонам, параллель
  через rayon, тепловая карта результата, устойчивость (соседние ячейки).
- **E5.3** Walk-forward анализ: скользящие окна in-sample/out-of-sample,
  агрегированный отчёт, детект переобучения.
- **E5.4** Расширенные метрики: Sortino, Calmar, MAE/MFE по сделкам,
  распределение P&L, drawdown-профиль, exposure; сравнение прогонов бок-о-бок.
- **E5.5 (S)** Монте-Карло по сделкам (перестановки/ресемплинг) → доверительные
  интервалы просадки.
- **E5.6 (S)** DSL/конфиг стратегий: описание стратегии данными (JSON: сигналы
  из индикаторного конвейера E3.3 + правила входа/выхода/риска) — новые
  стратегии без перекомпиляции.

### E6 — Опционы: от аналитики к торговле (S)

- **E6.1** Живая опционная доска MOEX ISS: цепочка страйков, IV/греки по
  каждому, открытый интерес; улыбка калибруется по живым котировкам (модели
  уже в `domain::options`).
- **E6.2** Анализ позиции: агрегированные греки портфеля во времени/по споту,
  payoff «сейчас» (по модели) и «на экспирацию», сценарии (спот ±X%, IV ±Y пп,
  время −N дней).
- **E6.3 (C)** IV-поверхность 3D и история IV Rank/Percentile.

### E7 — LLM-копайлот (S)

- **E7.1** Транспорт `data::llm` (Anthropic API, ключ в keyring): Key Activity
  summary на живых данных (промпт уже в `domain::keyactivity`), деградация в
  локальный fallback-свод без сети.
- **E7.2** Morning brief: автоматический утренний обзор — breadth-режим,
  секторные лидеры, аномалии FUTOI/HI2, календарь экспираций.
- **E7.3 (C)** Чат-панель над данными терминала: вопросы на естественном языке
  → вызовы IPC-инструментов (tool use) → ответ со ссылками на панели.

### E8 — Платформа и качество (M)

- **E8.1** Упаковка и обновления: MSI/NSIS-сборка в GitHub Actions (Windows
  runner), иконки, версия из Cargo, Tauri updater (подписанные релизы) — S.
- **E8.2** Производительность (см. NFR §5): виртуализация всех таблиц,
  батчинг событий Rust→UI (кадровая коалесценция ≤ 60 Гц), профилирование.
- **E8.3** Наблюдаемость: панель «здоровье» (статусы фидов, лаг, память, размер
  БД), структурированные логи в файл с ротацией, экспорт диагностики.
- **E8.4** Локализация RU/EN (словарь, переключатель в настройках) — S.
- **E8.5** Тёмная/светлая темы, горячие клавиши с редактором — S/C.

## 5. Нефункциональные требования (NFR)

| Метрика | Целевое значение |
|---|---|
| Обработка тиков (ядро, 1 инструмент) | ≥ 50 000 тиков/с без роста лага |
| Лаг «событие фида → пиксель» | p50 ≤ 100 мс, p99 ≤ 250 мс |
| Обновление DOM/ленты | ≤ 16 мс на кадр (60 FPS) при 20 уровнях × 10 инстр. |
| Старт приложения до интерактива | ≤ 3 с (тёплый) |
| Память при 100 инструментах в вотчлисте | ≤ 1.5 ГБ |
| Бэктест: 1 год минуток, 1 инструмент | ≤ 5 с; тики 1 месяц ≤ 30 с |
| Целостность истории | 0 молчаливых дыр (все зазоры видимы в отчёте качества) |
| Надёжность live-сессии | авто-восстановление фида без действий пользователя |
| Безопасность | секреты только keyring/.env; live-заявки — двойное подтверждение; логи без секретов |
| Кросс-платформенный CI | fmt + clippy -D warnings + все тесты зелёные на Linux |

## 6. Метрики успеха V3

1. Полный дневной цикл P1 (скрин → вход из DOM → сопровождение → журнал)
   выполняется без выхода из терминала, на живых данных Finam.
2. Квант-цикл P3 (история → стратегия → walk-forward → paper) — без ручной
   правки данных.
3. NFR §5 подтверждены бенчмарками в CI (perf-регрессии ловятся).
4. ≥ 90% строк `domain` покрыто тестами; интеграционные фикстуры для всех
   сетевых контрактов.

## 7. Этапность (мейлстоуны)

- **M1 — Live-данные и качество** (E1.1–E1.4): терминал живёт на реальном фиде.
- **M2 — Рабочее пространство** (E2.1–E2.4, E3.1–E3.3): docking, график,
  скринер, единые алёрты.
- **M3 — Торговля** (E4.1–E4.4, затем E4.5): DOM-торговля, риск, журнал,
  live-роутинг.
- **M4 — Исследования** (E5.1–E5.4): тиковый бэктест, оптимизация, walk-forward.
- **M5 — Order flow и опционы** (E2.5–E2.6, E6.1–E6.2).
- **M6 — Копайлот и релиз** (E7.1–E7.2, E8.1–E8.3, полировка S/C).

Зависимости: M1 → всё; M2 ∥ M1 (на моках); M3 требует M1; M4 требует E1.3;
M5 требует M1 + E2.1; M6 — последним.

## 8. Риски

| Риск | Митигация |
|---|---|
| Контракт ALGOPACK/ISS отличается от ожиданий | фикстуры до кода (`(verify)`-шаги первыми в M1) |
| Производительность WebView на heatmap/footprint | Canvas/WebGL + децимация в Rust; прототип-спайк до полной реализации (E2.5) |
| Live-роутинг: ошибки → реальные деньги | paper parity сначала, риск-движок обязателен, staged rollout, kill switch |
| Лимиты API (200 req/min) при бэкфилле | планировщик уже держит лимит; ночные окна, докачка постранично |
| Разрастание фронта | индикаторный конвейер и единый центр алёртов как общие ядра, не копипаста по панелям |
